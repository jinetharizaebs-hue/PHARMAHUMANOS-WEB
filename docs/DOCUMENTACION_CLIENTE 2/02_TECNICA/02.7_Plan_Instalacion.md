
**Script 7: Función para Actualizar Timestamps**
```sql
-- Función para actualizar automáticamente updated_at
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = CURRENT_TIMESTAMP;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Aplicar a tablas
CREATE TRIGGER update_productos_updated_at
  BEFORE UPDATE ON productos
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_clientes_updated_at
  BEFORE UPDATE ON clientes
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_facturas_updated_at
  BEFORE UPDATE ON facturas
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();
```

## 🔧 TROUBLESHOOTING

### Problemas Comunes y Soluciones

#### Problema 1: "Cannot connect to Supabase"

**Síntomas:**
- Aplicación no carga datos
- Error en consola: "Failed to fetch"

**Solución:**
```bash
# 1. Verificar variables de entorno
echo $VITE_SUPABASE_URL
echo $VITE_SUPABASE_ANON_KEY

# 2. Verificar que Supabase está activo
curl https://[tu-proyecto].supabase.co/rest/v1/

# 3. Revisar políticas RLS
# Ir a Supabase Dashboard → Authentication → Policies
# Verificar que existan políticas de acceso

# 4. Verificar CORS
# Dashboard → Settings → API → CORS
# Agregar dominio de producción
```

---

#### Problema 2: "Images not loading from Cloudinary"

**Síntomas:**
- Imágenes no se muestran
- Error 404 en URLs de imágenes

**Solución:**
```bash
# 1. Verificar credenciales de Cloudinary
console.log(import.meta.env.VITE_CLOUDINARY_CLOUD_NAME)

# 2. Verificar formato de URL
# Debe ser: https://res.cloudinary.com/[cloud-name]/image/upload/...

# 3. Verificar que imagen existe en Cloudinary
# Ir a Media Library → Buscar imagen

# 4. Verificar transformaciones
# URL debe ser válida, ej:
# /image/upload/w_800,h_800,c_limit/productos/abc123.jpg
```

---

#### Problema 3: "Build fails on Vercel"

**Síntomas:**
- Deploy falla
- Error: "Build exceeded maximum duration"

**Solución:**
```bash
# 1. Verificar que build funciona localmente
npm run build

# 2. Revisar logs en Vercel
# Dashboard → Deployments → [deployment] → View Logs

# 3. Verificar node_modules no esté en Git
# .gitignore debe incluir:
node_modules/
dist/
.env.local

# 4. Limpiar caché de Vercel
# Settings → General → Clear Build Cache

# 5. Re-deploy
vercel --prod --force
```

---

#### Problema 4: "Users can't login"

**Síntomas:**
- Credenciales no funcionan
- Error: "Invalid credentials"

**Solución:**
```bash
# 1. Verificar que usuario existe en Supabase
# Dashboard → Authentication → Users

# 2. Si usa email/password, verificar confirmación de email
# User debe tener "Email Confirmed" = true

# 3. Resetear password
# Dashboard → Authentication → Users → [usuario] → Reset Password

# 4. Verificar políticas RLS no bloqueen autenticación
```

---

#### Problema 5: "Stock not updating"

**Síntomas:**
- Factura se crea pero stock no cambia
- Movimientos de inventario no se registran

**Solución:**
```sql
-- 1. Verificar trigger existe
SELECT * FROM information_schema.triggers 
WHERE event_object_table = 'facturas';

-- 2. Verificar función actualizar stock
-- Revisar código en InvoiceScreen.jsx

-- 3. Verificar manualmente
SELECT id, nombre, stock FROM productos WHERE id = [producto_id];

-- 4. Verificar logs de movimientos
SELECT * FROM movimientos_inventario 
WHERE producto_id = [producto_id] 
ORDER BY created_at DESC 
LIMIT 10;
```
